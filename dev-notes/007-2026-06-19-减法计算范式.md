## 1 · 哲学定位 · 米开朗基罗的雕塑

米开朗基罗对自己的雕塑说过一段流传的话:

> I saw the angel in the marble and carved until I set him free.
> (我看见天使藏在大理石里 · 我雕刻 · 直到把他释放出来。)

这是 **subtractive sculpture** (减法雕塑) 的哲学:**艺术家不是在添加什么 · 是在移除多余的部分 · 让作品显形**。

跟之相对的是 **additive sculpture** (加法雕塑) · 比如黏土塑形 / 3D 打印 · 一点点添加材料构建作品。

**减法计算范式** 提议:

> 算法的真正力量 · 不在它执行了什么 · 而在它**没执行**什么。
> 答案不是 build 出来的 · 是从所有可能 state 里 carve 出来的。
> 大部分计算 "would have happened" 但物理上没真发生 —— 这就是 counterfactual。

## 2 · 传统范式 · 加法计算 (Additive Computation)

### 核心隐喻

building / accumulating · 添加 / 累积

### 算法学家做什么

- 设计 "在每一步做什么操作 (what TO DO)"
- state 从 initial 通过 operations 逐步加工到 final
- 优化目标:减少 operation 步数

### state 演化

```
input ──▶ op1 ──▶ op2 ──▶ op3 ──▶ ... ──▶ output
        每步 op 都执行 · 都付钱 · 都改变 state
```

### Cost 模型

- 每个 operation 都有代价 (时间 / 空间 / oracle 调用 / 能量)
- 总代价 = 所有执行 operations 的成本之和
- 越少 operation 越好

### 典型问题表达

"input → process → output"

### 失败处理

- operation 失败需要 retry / backtracking
- 失败也有成本 (重做的成本)

### 代表性算法

经典并行 / Grover / 标准 quantum query · 大部分主流算法 · 包括 ML 训练 / 编译器 / 数据库查询。

## 3 · 新范式 · 减法计算 (Subtractive Computation)

### 核心隐喻

**sculpting / pruning** · 雕塑 / 剪枝 / 让答案显形

### 算法学家做什么

- 设计 "什么不要做 (what NOT TO DO)" · 让 "no-trigger" 自然形成 answer
- 所有可能 state 在 superposition 同时存在 · 通过 counterfactual probes 筛掉不要的
- 留下来的就是答案

### state 演化

```
all-possible-states (在 superposition 中并存)
     │
     ▼ Φ_f1 (filter 1)   触发率 δ_1 ≈ 0
     │
     ▼ Φ_f2 (filter 2)   触发率 δ_2 ≈ 0
     │
     ▼ Φ_f3 (filter 3)   触发率 δ_3 ≈ 0
     │
     ▼
surviving state = answer
(大部分 state 被 "无形地" 筛掉 · 物理上没付钱)
```

### Cost 模型

- 只为 "would-have-fired" 的 operation 付钱
- 大部分 operation 物理上不发生 (反事实)
- 总代价主要由 surviving items 数量决定 · 不由总尝试次数决定

### 典型问题表达

"all possibilities → counterfactual filter → surviving = answer"

### 失败处理

- "failure" 在物理上不发生 (probe 没触发)
- 没有重试代价
- 失败的分支自动 "vanish" · 不留痕迹

### 代表性算法

`dev-notes/008` 列出的 6 个 (CPA · CBB · CAL · CV · CA* · CGTS) · 都是减法范式的实例。

## 4 · 关键对比表

| 维度 | 加法计算 | 减法计算 |
|---|---|---|
| 隐喻 | 建造 / 累积 | 雕塑 / 剪枝 |
| 算法学家关注 | what TO DO | what NOT TO DO |
| state 起点 | 单个 initial | 所有可能并存 |
| 演化方式 | 逐步 transform | 筛除不符合的 |
| 答案位置 | 最终 state | 幸存 state |
| 失败成本 | 高 (要重做) | 0 (没真做) |
| 主要 cost driver | 操作次数 | 幸存项数 |
| 不可见操作 | 没意义 (没做) | 是核心 (counterfactual) |
| 适合问题 | 多数 (default) | 稀疏 + 副作用 + verify-without-commit |

## 5 · 减法范式的杀手锏问题类

### 类 K1 · 几乎所有候选都该被淘汰 (sparsity)

- 例:$10^9$ 个分子里找 binding affinity > threshold 的
- 加法成本:测 $10^9$ 次
- 减法成本:并行反事实测全部 · 真测的只有少数 hit · ~$O(K)$ where $K \ll N$

### 类 K2 · 评估有不可逆副作用 (side effects)

- 例:测试某 system 是否能 jailbreak · 测试本身可能触发 alert / 启动防御
- 加法成本:每次测都暴露 · 测 N 次暴露 N 次
- 减法成本:反事实测 · 大部分物理上没发生 · 不留痕

### 类 K3 · 需要 verify 但不想 commit (verify-without-commit)

- 例:加密 token 是否有效 · 但用了就 burn
- 加法成本:用了才知道 · 没用没法知
- 减法成本:反事实验证 · 知道有效性但不消耗

### 这 3 类的共性

都涉及**queries 有物理代价** · 而不是 query 本身复杂度高。**减法范式的核心是把"查询代价"重新定义为"实际发生的查询代价" · 而不是"所有尝试查询的代价"**。

## 6 · 跟其他非传统范式的关系

减法计算不是孤立提出 · 跟若干已有范式有亲缘:

| 范式 | 跟减法计算的关系 |
|---|---|
| **Probabilistic computing** (Monte Carlo / SAT) | 都接受随机性 · 但概率算法仍是加法 (每个 sample 是真执行的) · 减法是反事实 |
| **Approximate computing** | 都接受不精确 · 但近似算法仍真执行所有 op (只是结果不精确) · 减法是不真执行 |
| **Lazy evaluation / call by need** | 都"延后执行" · 但 lazy 是 "用到才执行" · 减法是 "用到也不一定真执行" |
| **Speculation in CPU** | 都"先假设再修正" · 但 speculation 真执行了 · 失败时回滚 · 减法是物理上没执行 |
| **Reversible computing** (Bennett) | 都关心"操作的可撤销性" · reversible 是 logical 可逆 · 减法是 physical 不发生 |
| **Quantum amplitude amplification** | Grover 是加法 (每次迭代真触发) · 减法版本是反事实 Grover (大部分迭代不触发) |
| **Monte Carlo tree search** (MCTS) | rollout 是真执行 · 减法版 CGTS 是反事实 rollout |

减法范式跟它们的差异化在 R1+R2+R3 三个性质同时具备 (见 `dev-notes/004 §4`)。

## 7 · 哲学意义 · "计算的本体论"

加法范式默认:**计算是发生在物理世界的事件** · 每个 op 都是一次物理操作 · 都消耗资源 · 都留痕迹。

减法范式提出:**计算可以发生在"潜在事件"层面** · 大部分 op 没真物理发生 · 但算法的输出依赖于它们 would-have-done。这跟物理学的 **counterfactual definiteness** 哲学问题深度相关 (Bell 不等式 / EPR / Wheeler 哲学全部围绕这点)。

减法计算的工程实现 (CFE 算子) 把这个哲学问题变成可工程化的实际能力。这是计算机科学跟物理基础研究的少有的真接触点。

类比:**Microservices 之于 monolith** —— 不只是"小一点的服务" · 而是**计算的部署粒度本体变了**。
减法范式之于加法范式 —— 不只是 "更少操作的算法" · 而是 **计算的"事件" 本体变了**。

## 8 · 用减法范式重新看 5 个经典问题

| 问题 | 加法视角 | 减法视角 |
|---|---|---|
| Search (在 N 中找 hit) | Grover · $O(\sqrt{N})$ 真查询 | 反事实 OR · 测 "有没有 hit" $O(\sqrt{N}/\delta)$ 真查询期望 $\ll \sqrt{N}$ |
| Verification (token 是否有效) | 用一次 · 消耗 | 反事实验证 · 不消耗 · 后续仍可用 |
| Multi-assay (N 个样本测属性) | 每样本测 · 销毁 N 个 | 反事实测全部 · 物理上销毁 0 个 · 只统计结果 |
| Adversarial probe (对手有没有 trap) | 暴露探测者 | 反事实探测 · 对手物理上看不到 |
| Active learning (标 N 候选要 label 谁) | 选 informative 的 · 付 label cost | 反事实预筛 · label cost 0 · 只对 surviving 真 label |

每个问题都从 "怎么少做几次操作" 变成 "怎么让大部分操作物理上不发生"。**问题没变 · 范式变了 · 解法变了**。
