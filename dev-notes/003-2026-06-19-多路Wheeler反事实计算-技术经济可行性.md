## 2 · 工业可行性 · 三个成熟度阶段

### 阶段 1 · 利基商品(已经在卖)

**IFM/counterfactual 的真实商业雏形已经存在 · 但不在"计算" · 在"成像/探测"**:

- **Undetected photons imaging** · QuantIC consortium 衍生 / Quantum Optical Systems 等在卖
- **客户**:制药 R&D(光毒性敏感样本)、显微镜厂商、半导体晶圆检测、mid-IR 分子识别
- **价值主张**:同样信息量 · 光强降低 10-100 倍 · 减少 photo-damage

**真正可能在这阶段加入的 niche**(基于今天 lab 能力):

| niche | 客户 | 竞争 | 出货可能性 |
|---|---|---|---|
| Counterfactual 生物成像(光敏样本) | 制药 / 显微镜 | 共聚焦 / 双光子 | 高 · 已有商业雏形 |
| 低光通量 mid-IR 分子光谱 | 海关 / 反恐 / QC | FTIR | 中高 · 差异化场景 |
| 高敏感测试(不许触发被测系统) | 网络安全 / 国安 | 无直接竞争 | 低-中 · niche 真 · 客户少 |
| 多 obstacle "并行 OR 计算"产品 | ??? | 经典并行 | **极低** · 客户故事不成立 |

### 阶段 2 · 平台级(需要突破集成度)

需要解决:

- **N 从 12 → 100+** · 当前 5 dB loss 在 N=100 累积成 ~50 dB(完全不可用)· 需要新材料平台或新架构
- **Active phase 稳定化** · 多模相位漂移在大 N 下不可控
- **室温探测器** · SPAD 替代 SNSPD · 效率从 90% 掉到 30-50%
- **跟经典系统接口延迟** · 必须 < ms 才能融入工作流

**进入这阶段需要 platform-level 投资**(类似 PsiQuantum / Xanadu 规模)· 不是个人/小团队能做。

### 阶段 3 · 通用计算(不可达)

不要寄希望。Bomb query 复杂度 B(f) = Θ(Q(f)²) 比标准量子查询慢平方 · 也不优于经典 · **这条线物理上不可能成为通用计算范式**。

**结论**:工业可行 = 阶段 1 niche 商品(成像/探测主导)+ 阶段 2 平台级有可能但要平台投资 + 阶段 3 排除。商业窗口在"成像/测试/探测",不在"计算"。

## 3 · 能加速什么计算 · 算法分类

### 类 A · 已有理论加速(标准量子贡献 · 非 bomb 贡献)

- NAND-tree 评估:Farhi 2008 · O(√N) vs 经典 Ω(N^0.753)
- Boolean formula 评估:Reichardt 2010 推广 read-once formula
- Element distinctness · Collision finding 等

→ 这些**能在 N-path bomb interferometer 上跑** · 但加速来自标准量子查询 · bomb 反而是 ×2 negative · **没有 bomb 独有贡献**。

### 类 B · 真正利用 counterfactual 性质(物理不可替代 · 这是真 niche)

- 测试 oracle 是否会响应 · 但不让它响应:
  - honeypot/canary 是否被触发 · 而不再触发它
  - rate-limited API 的 query 是否返回非空 · 而不消耗 quota
  - one-time token 是否仍有效 · 而不烧掉它
- 测试稀缺资源:光敏样本 / 易碎品 / 一次性实验
- 测试 quantum state:探测寄存器内容而不导致 decoherence(量子计算自身的辅助技术)
- Adversarial probe:测试对手是否有蜜罐/监听 · 而不暴露探测行为

**这是唯一**经典+标准量子都做不到的类 · 应用价值高度依赖具体场景能否物理 encode。

### 类 C · 看似能用但是陷阱(必躲)

- "并行 SAT 求解":每个 obstacle 需要物理编码 · 编码成本随问题规模线性增长 · 没有 net advantage
- "数据库搜索":同样编码成本陷阱 · Grover 已有更好理论
- "图算法":量子游走+Grover 更成熟

→ **别做** · 投资人/客户被说服后会被技术 DD 撕碎。

## 4 · 问题重构 · 最难也最值钱的一步

### 4 个重构模板 + 它们的陷阱

#### 模板 1 · 销毁性测试 → bomb query

- **原问题**:N 个 sample · 想知道有没有 defective · 测试销毁 sample
- **重构**:每个 sample 编码为一条光路的 obstacle (defective = bomb) · 单 photon 通过 interferometer · 看是否干涉
- **优**:大部分 run 不实际测试 sample · 单光子 N 路检测
- **陷阱**:photon-sample 物理 interfacing · 每个 sample 怎么跟一条 chip waveguide 耦合 · 是关键工程难题
- **适用**:微样本生物 / 易碎晶体 / 半导体晶圆缺陷探测

#### 模板 2 · 配额受限服务 → bomb query

- **原问题**:经典 API 按次收费 · 想知道 N 个潜在 query 有没有 hit · 只想付 hit 的钱
- **重构**:N 个 query 编 N 条光路 · 每条上 bomb iff query 会 hit · IFM 得 "any hit?"
- **优**:理论上节省 API 调用
- **陷阱(致命)**:API 不是 quantum oracle · 没法用 photon 探测远程经典 server · 必须 emulate server 行为到量子电路 · 这一步成本就 > 直接调 N 次
- **适用**:**不适用** · 经典数字服务无法 bomb-query · 别陷进去

#### 模板 3 · adversarial probing → bomb query(真 niche)

- **原问题**:对手部署 N 个 honeypot/trap · 想知道有没有 trap · 触发任何一个就暴露
- **重构**:N 个目标位置编码为 N 路 IFM probe · quantum-correlated probe photon 与目标物理耦合 · counterfactual 推断 trap 存在
- **优**:对手观察不到我的探测 · 唯一能做的物理范式
- **陷阱**:trap 本身必须能被 photon/微波 一致编码为 obstacle · 数字 trap 不行 · 物理 trap (传感器/摄像头/电磁监控) 可以
- **适用**:军事情报 / 物理入侵检测 / 物理传感网络 probing · niche 真存在但客户极少

#### 模板 4 · 状态测量 → bomb query(量子内部应用)

- **原问题**:量子计算机长 sequence 计算 · 想中途偷看 qubit · 但偷看 = decoherence
- **重构**:ancillary photon 做 IFM · 不实际跟主 qubit 纠缠 · 推断主 qubit 是 0 还是 1
- **优**:真应用 · 量子计算机自己需要非破坏性测量
- **陷阱**:跟主流 quantum error correction 重叠 · 不一定优于 stabilizer measurement
- **适用**:counterfactual 的真亲家 · 量子计算辅助技术 · 客户是其他量子计算机厂商

### 元规则 · 3 条筛选

满足全部 3 条才适合 bomb query 重构:

1. **目标系统本身支持 quantum probe**(光/微波/自旋的物理耦合 · 不能是纯数字/远程/抽象 systems)
2. **触发代价远大于探测投资**(否则不如直接触发)
3. **"any of N" 单 bit 结论必须有用**(否则需 Grover-like O(√N) 迭代定位 · 效率退化)

### 同时满足 3 条的真实 niche

- **半导体晶圆 wafer 缺陷 IFM 检测** · sample 真物理 + photo-damage 是真代价 + 单 bit "any defect" 够用(然后转 destructive 精细测试)
- **光敏药物/蛋白/细胞 IFM 显微** · 真物理样本 + photo-bleaching 真代价 + "any abnormal feature?" 足以筛选
- **生物威胁物 detection**(爆炸物/神经毒素)· 真物理 + 触发危险 + "any threat present?" 单 bit 够触发 evacuation
- **军用 standoff 传感** · 探测对手雷达/激光 trap 是否激活 · 不被察觉 · 单 bit "any active sensor" 足以决策

**全部是成像/探测 niche · 不是"计算"** · 跟工业可行性段的阶段 1 商业窗口完全吻合。
