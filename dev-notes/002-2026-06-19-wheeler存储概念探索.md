## 0 · Wheeler 教给我们的存储启示

Wheeler 实验的核心**不是**"光是波或粒子" · 而是:

> **reality is information-resolved at measurement time** · 物理实在的"内容"由测量时机和方式决定 · 而不是写入时机

这条原则套到存储上有非平凡的含义:

- 现有所有存储(DRAM / Flash / 磁带 / DNA / 相变 / 量子 RAM / EIT 慢光 / 稀土晶体)都遵守 "**写时确定 · 读时检索**"的范式
- 现有量子存储把"量子"用在 (a) 更高密度 (Holevo bound) 或 (b) 更难复制(no-cloning · quantum money)
- **没人**把延迟选择那条"**测量基决定语义**"作为存储 primitive 暴露给应用层

真正的新意不在"用光做经典存储更快" · 在于:

> **存储介质本身模糊 · 读时才坍缩成具体值** · 跟所有现有存储**正交**

## 1 · 元判据:什么算"革命性 niche"

筛选时优先看:

A. **唯一性**:这个 niche 有没有任何经典 / 软件 / 现有量子方案能等价解决?如果有 · 不算革命
B. **市场实在性**:niche 有真实付费需求吗?(不需要大 · 但要真)
C. **落地路径**:从概念 → Blender demo → 硬件 PoC → 产品 · 是否有可见路径
D. **跟 Wheeler 物理的本质连接**:是否非要 Wheeler 原理才能实现 · 还是 Wheeler 只是表面装饰

## 2 · 候选清单

### 概念 1 · 薛定谔的字 / Choice-Resolved Storage

- **做什么**:一个存储单元物理上同时存着 X 和 Y 两个不同的经典值。读时选基 A 得 X · 选基 B 得 Y · 读了一种另一种永久消失 (no-cloning)
- **物理实现**:把信息编码在 photon 的 polarization × path 联合自由度 · 测 polarization vs 测 path 得不同投影
- **niche**:**完全 oblivious 的查询服务**
  - 数据持有方(server)不知道客户端会查什么
  - 客户端拿到答案后服务端也不知道客户端查的是哪种
  - 现有 PIR (Private Information Retrieval) 用密码学多 round 通信 · 带宽 / 计算开销巨大
- **为什么革命**:把"保密查询"从软件协议层下沉到物理介质层 · 信任锚从"数学难题"(LWE / DDH)变成"测量塌缩" · 服务端就算被完全控制也只能给一种答案
- **demo 化**:Blender 里两条偏振路径 + 两组探测器 · 用户选基 → 看到不同结果 · 另一答案在原地"塌缩消失"
- **落地难度**:中 · 单 photon polarization + path 复合编码已有实验

### 概念 2 · 量子擦除审计内存 / Tamper-Evident Memory ⭐ 推荐

- **做什么**:存数据 + which-path tag
  - 任何**未授权读取**必须先 query tag → 留下塌缩痕迹 · 永久可验
  - **授权读取**先擦除 tag → 干涉读出 + 同样留痕但带授权签名
  - **没有沉默读 (silent read) 的物理可能性**
- **物理实现**:photon 双路径 + 每路偏振 tag · 读时要么直接探测 tag (which-path · 留 trace),要么先过 45° eraser 再干涉测 (recover coherence · 也留 trace)
- **niche**:**强监管行业的审计日志**
  - HIPAA 医疗 / GDPR 隐私 / SOX 财务 / PCI-DSS 支付 / 国家安全级机密
  - 现行方案靠 OS / DB 记访问 log · 但 root / DBA 能改 log
  - Wheeler 审计内存让"读取行为本身"成为物理可证事件 · root 也改不了过去的塌缩
- **为什么革命**:把"读访问的审计"从 OS / DB 层降到物理层 · **第一次让 "silent read 不可能" 成为物理定律而不是 IT 政策**
- **demo 化**:在现有 `wheeler_demo.py` 基础上加偏振 tag 阶段 + eraser 阶段 · 演示
  - 场景 A:adversary 偷读 → 探测器留塌缩痕 · 干涉消失
  - 场景 B:authorized 读 → eraser 先 + 干涉 + 签名痕
  - 场景 C:silent read 尝试 → 物理上不存在能不留痕的读取路径
- **落地难度**:低-中 · 偏振 tag + eraser 是成熟光学 · 难点在"塌缩痕"的不可伪造审计协议

### 概念 3 · 反事实存储 / Counterfactual Bit Storage

- **做什么**:用 Elitzur-Vaidman bomb test / 量子 Zeno 效应 · 信息编码在 "**没有发生的事件**" 里
  - 读取时某些路径上 photon 没出现 → bit 是 1
  - **物理上零光子穿越存储介质**
- **物理实现**:Salih et al. 2013 counterfactual communication protocol 的变种 · interaction-free measurement
- **niche**:
  - **深空 / 高辐射环境** (探测器 / 卫星 / 核反应堆内 sensor) · 任何物理介质都会被辐射损伤 · 但"没光子流过"的介质本身就少受损
  - **极致 tamper-evidence** · 截获改变 absence · 监听者没 signal 可截
- **为什么革命**:存储不再需要"持有物理 bit" · 突破"物质是信息载体"的预设 · 载体是 path 上的概率分布 · 不是任何粒子
- **demo 化**:Blender 里画 Mach-Zehnder + bomb (吸收体) · 演示干涉条件 vs 触发条件 · 视觉抽象 (要看的是 "不存在的光" · 反直觉)
- **落地难度**:高 · 概念门槛 + 工程精度都难 · counterfactual 协议效率低

### 概念 4 · 时间延展存储 / In-Flight Memory with Deferred Schema

- **做什么**:photon 在长光纤 loop 里循环 (μs-ms 级停留 · 现有 EIT / WGM 已实现)
  - tap 点决定读出方式:插入 BS2 → 干涉值 A · 不插 → which-path 值 B
  - 两个值在 write 时同时编码
- **物理实现**:光纤 delay line / whispering-gallery resonator + 可切换 BS2
- **niche**:**路由器内多角度 data buffer**
  - 同一个数据包对路由器是地址 · 对监控是 ID · 对调度是优先级
  - Wheeler buffer 让一个 photon 同时承载 N 种解读 · 每个下游消费者按需取
- **为什么革命**:存储容量不变 (Holevo bound 限定 1 photon = log d 经典 bit) · 但 "schema 选择" 从 write 时迁移到 read 时 · 完全消除"读不同 schema 要重新 buffer"的副本浪费
- **demo 化**:Blender 里光环 loop + tap + 可切换 BS2 · 视觉直观 (看光在转圈)
- **落地难度**:低 · EIT / WGM 是成熟技术 · 主要是控制电路设计

## 3 · 私心想推但没选的 3 个

| 方向 | 一句话 | 为什么没选 |
|---|---|---|
| 可逆存储 (Reversible Storage) | 写 / 反写互为算子 · 趋近 Landauer 能量极限 (kT·ln2/bit) | 跟 Wheeler 关系薄 · 是 Bennett 可逆计算路线 · 不能算"Wheeler-inspired" |
| Photon-as-pointer | 存的不是值而是引用 · 读时跟一个共享池叠加 | 概念美但 niche 模糊 · 像在追求"光做指针"的形式美感 · 缺杀手应用 |
| 纠缠分布存储 | 数据在 Alice + Bob 之间分布存 · 单方读破坏 · 必须协作 | 是 quantum secret sharing 已有研究 · 不够新 |

## 4 · 推荐 · 概念 2 (量子擦除审计内存)

按权重:

1. **niche 是 $10B+/年真实市场** · HIPAA / GDPR / SOX / PCI-DSS / 国安级审计 log · 没 PoC 也能讲投资人 / 学术故事
2. **唯一性 max** · silent read 不可能是物理定律而不是政策 · 没有任何经典 / 软件方案能给同样承诺
3. **demo 化最容易** · 在现有 `wheeler_demo.py` 基础上加 1 偏振 tag 阶段 + 1 eraser 阶段 · 直接可视化"sneak read 留下不可抹去的痕迹"
4. **跟现有量子密码学不重复** · QKD 做传输安全 · 这条做**存储 + 读取的可审计性** · 没人系统做过
5. **可分层落地** · L1 概念演示 (Blender) → L2 OPM (optical pulse memory) 硬件 PoC → L3 集成到 SIEM 的物理存储后端

**次推**:概念 1 (薛定谔的字 · oblivious search) · 因为 PIR 是真问题 · 但工程上离落地远
**不推**:概念 3 (反事实存储) · 太前沿 · 概念美但 niche 模糊
**不推**:概念 4 (in-flight buffer) · niche 有但跟现有 EIT 存储增量小 · 不够革命
