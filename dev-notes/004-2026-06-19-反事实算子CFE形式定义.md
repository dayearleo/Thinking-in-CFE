## 1 · 数学定义

### 1.1 · 基础设定

设:

- $X = \{0,1\}^N$ 是 N 维 Boolean 输入空间
- $\mathbf{x} = (x_1, \ldots, x_N) \in X$ 是一个具体输入 · 不直接可访问 · 只通过 oracle
- $O_i$ 是第 $i$ 个标准 quantum oracle:
$$O_i : |b\rangle \otimes |a\rangle_{\text{anc}} \;\mapsto\; |b\rangle \otimes |a \oplus x_i\rangle_{\text{anc}}$$
- $\hat{O}_i$ 是 $O_i$ 的 **counterfactual 变种**:做"是否会触发"的探测 · 但实际触发概率被压到 $\leq \delta$
- $f: \{0,1\}^N \to \{0,1\}$ 是任意 Boolean 函数

### 1.2 · 核心定义

**定义 1 (Counterfactual Function Evaluation Operator · $\Phi^{CF}_f$)**

对任意 Boolean 函数 $f$ · 算子 $\Phi^{CF}_f$ 定义为映射:

$$\Phi^{CF}_f : (\hat{O}_1, \ldots, \hat{O}_N) \times |\psi_0\rangle \;\longrightarrow\; y \in \{0,1\}$$

满足以下三条性质:

**(P1) 正确性**:
$$\Pr\left[y = f(\mathbf{x})\right] \geq 1 - \epsilon$$

**(P2) 反事实性**:对每个 $i$ 满足 $x_i = 1$:
$$\Pr\left[O_i \text{ 在求值期间被物理触发}\right] \leq \delta$$

**(P3) 代价上界**:实现 $\Phi^{CF}_f$ 需要的总 oracle 调用次数为:
$$B_\delta(f) = O\left(\frac{Q(f)^2}{\delta}\right)$$

其中 $Q(f)$ 是 $f$ 的标准 quantum query complexity (Lin & Lin 2015 CCC)。

### 1.3 · 极限与退化

| 参数取值 | 行为 |
|---|---|
| $\delta \to 0$ | 完全无触发 · 但 $B(f) \to \infty$ · 实际工程取 $10^{-2}$ 到 $10^{-6}$ |
| $\delta = 1$ | 退化为标准 quantum query · $B(f) = Q(f)^2$ · 失去反事实优势 |
| $N=2, f=\text{NOR}$ | 还原为 Elitzur-Vaidman bomb tester (1993 原始版本) |
| $N \geq 2, f=\text{NOR}$ | Wheeler 多路推广(本项目 user 提出 reframe 的核心形态) |
| $f = \text{NAND-tree}$ | Farhi-Childs NAND-tree algorithm 在 bomb query 模型下的版本 |

## 2 · 子算子目录(具体 $f$ 实例化)

| 算子 | $f$ 形式 | 语义 | 代价上界 $B_\delta(f)$ |
|---|---|---|---|
| $\Phi^{CF}_{\text{OR}}$ | $\bigvee_i x_i$ | 至少一个 $x_i = 1$? | $O(\sqrt{N}/\delta)$ |
| $\Phi^{CF}_{\text{AND}}$ | $\bigwedge_i x_i$ | 全部 $x_i = 1$? | $O(\sqrt{N}/\delta)$ |
| $\Phi^{CF}_{\text{NOR}}$ | $\neg\bigvee_i x_i$ | 全部 $x_i = 0$? (Wheeler 推广原型) | $O(\sqrt{N}/\delta)$ |
| $\Phi^{CF}_{\text{MAJ}}$ | majority | 多数为 1? | $O(N/\delta)$ |
| $\Phi^{CF}_{\text{COUNT}}$ | $\sum_i x_i$ | 多少个 1? | $O(\sqrt{N \cdot \text{ans}}/\delta)$ |
| $\Phi^{CF}_{\text{T}_t}$ | threshold $\geq t$ | 至少 $t$ 个为 1? | $O(\sqrt{Nt}/\delta)$ |
| $\Phi^{CF}_{\text{LOC}}$ | locate first 1 | 哪个 $i$ 是第一个 1? | $O(\sqrt{N}/\delta)$ 期望 |
| $\Phi^{CF}_{\text{NAND-tree}}$ | balanced NAND-tree | 平衡 NAND-tree 评估 | $O(\sqrt{N}/\delta)$ |

## 3 · 算法层接口规范

算法学家不关心物理 · 只关心调用接口。算子签名:

```
counterfactual_eval(
    f       : Predicate[Vec[Bit]] -> Bit,    // 要算的函数
    probes  : Vec[Oracle],                   // N 个 oracle 句柄
    epsilon : Float = 1e-3,                  // 允许错误率
    delta   : Float = 1e-3,                  // 允许触发率上界
) -> Result[Bit, OperatorError]
```

### 3.1 · 行为契约

- 返回值 $y$ 满足 $\Pr[y = f(\mathbf{x})] \geq 1 - \epsilon$
- 每个被探测的 oracle 触发概率 $\leq \delta$
- 接口暴露 $B_\delta(f)$ 作为可估算成本 · 调用者可预算
- 失败模式:如硬件保真度不足 / loss 累积 · 返回 `Err(UNRELIABLE)` 而非错误结果
- 可组合:算子可嵌套 · `Φ(g, [Φ(f_i, ...) for i in N])` 合法

### 3.2 · 跟其他原语调用对比

```python
# 经典 (必触发 · 全部留痕)
result = any(probe() for probe in probes)        # 触发 N 次

# 标准量子查询 (Grover · 必触发但少次)
result = grover_or(probes)                        # 触发 sqrt(N) 次

# CFE 算子 (大部分 run 不触发)
result = counterfactual_eval(any_, probes, delta=1e-3)
# 触发率 < 0.1% per probe · 大部分 run 无任何留痕
```

## 4 · 算子的本质性质 · R1 R2 R3

算子要立 · 必须有别的算子不具备的性质 · 否则只是已有算子的语法糖。$\Phi^{CF}_f$ 同时具备:

**R1 · 触发率任意小**:$\delta$ 是参数 · 可调
**R2 · adversary 不可检测**:在 $\delta \to 0$ 极限下 · 持有 oracle 的对手物理上无法察觉算子被执行
**R3 · 输入不消耗**:多次评估同一组 oracle 不损耗它们 (若为稀缺/易损资源)

### 跟其他算子的差异化对比

| 性质 | 经典 OR/AND | Grover/标准量子 | $\Phi^{CF}_f$ |
|---|---|---|---|
| 函数任意性 | ✅ | ✅ | ✅ |
| 输入并行 | ⚠️ 需 N 线程 | ✅ 叠加并行 | ✅ N 路同时探测 |
| 查询复杂度 | $N$ | $\sqrt{N}$ | $\sqrt{N}/\delta$ |
| **触发率上界 (R1)** | 100% | ~100% | ✅ 任意小 $\delta$ |
| **adversary 不可检测 (R2)** | ❌ | ❌ | ✅ |
| **输入不消耗 (R3)** | ❌ | ❌ | ✅ |
| 物理 oracle 要求 | 任意 | quantum | quantum (光/微波/spin) |
| 可组合 | ✅ | ✅ | ✅ |

**只有 $\Phi^{CF}_f$ 同时拥有 R1 + R2 + R3** —— 这是 differentiator 的本体。

## 6 · 限界 · 算子不是什么

为防止后续问题域代数化时滥用 · 必须明确算子**不能做什么**:

- **不能查询非物理 oracle** · oracle 必须有光/微波/spin 的物理实现 · 不能是远程 API / 抽象数据
- **不能消除单 bit 输出的局限** · $f$ 输出 $k$ bit 需要 $k$ 次独立调用 · 总成本 $k \cdot B_\delta(f)$
- **不能算 super-polynomial 的 $f$** · 如果 $Q(f)$ 已经指数 · $\Phi^{CF}_f$ 还是指数
- **不能突破 Holevo bound** · 从 $\Phi^{CF}_f$ 提取的信息量上界跟 standard quantum query 相同
- **不替代量子计算** · 算子针对 "查询代价" 问题 · 不是 "计算复杂度" 问题
- **不在 NISQ era 立即工程化** · 需要稳定多模 photonic chip + 高 fidelity probes · 当前 SOTA 限 $N \leq 12-32$
